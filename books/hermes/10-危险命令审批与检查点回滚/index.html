十、危险命令审批与检查点回滚（待验证）

结论先行：Hermes 提供两套独立的安全防线。执行前：tools/approval.py 对终端命令进行正则匹配，命中危险模式则阻塞等待用户审批，支持 CLI 交互输入和 Gateway 按钮审批两种方式。执行后：tools/checkpoint_manager.py 在每次文件写入前通过影子 git 仓库创建快照，用户随时可回滚到任意历史版本。两者联动：审批系统防止危险命令执行，检查点系统兜底文件误改。

10.3 CLI 审批交互

四种结果：

  • once：本次执行，下次同类命令仍需审批
  • session：本次会话内该模式免审批（存内存，重启失效）
  • always：写入 ~/.hermes/config.yaml command_allowlist，永久免审批
  • deny：拒绝执行，返回错误给 LLM
