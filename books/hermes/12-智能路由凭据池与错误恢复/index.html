十二、智能路由、凭据池与错误恢复(待验证)

结论先行：Hermes 的可靠性由三层机制保障。智能路由：简单对话自动降级廉价模型，降低成本；凭据池：多 API Key 轮换 + OAuth 自动刷新，确保高并发下不断线；错误分类与重试：15 类错误精确分类，对症下药，而非无脑重试。三者协作，让 Hermes 在 API 限速、Key 耗尽、网络抖动等场景下自愈。

    B -->|复杂消息/含代码/关键词| D[主模型 e.g. claude-opus]


    C --> E[CredentialPool.current、（、）]

12.6 错误→恢复动作映射表

错误类型 HTTP 恢复动作
rate_limit 429 抖动退避 + 轮换 Key（冷却 1h）
billing 402 立即轮换 Key（停止重试当前 Key）
auth 401/403 刷新 OAuth Token，失败则轮换 Key
context_overflow 400 触发上下文压缩，不轮换 Key
timeout - 重建 HTTP 客户端，退避重试
overloaded 503/529 退避重试（不轮换，服务商过载）
server_error 500/502 退避重试
model_not_found 404 切换备用模型/Provider
thinking_signature 400 清除 thinking block，重试
